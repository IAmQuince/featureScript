# CFD Screening Metrics

The GUI and CLI now compute first-pass geometry metrics to help decide whether a tributary layout is worth moving forward to CAD regeneration, meshing, and CFD.

These metrics are **not** a CFD solution. They are lightweight geometric proxies intended for comparison and debugging.

## Where to find them

Generated by the GUI export bundle or CLI:

```text
cfd_metrics_summary.txt
cfd_metrics.json
cfd_branch_metrics.csv
```

Main implementation:

```text
src/tributary_geometry/metrics.py
```

## Metrics included

Summary metrics include:

- branch count
- circle count
- total centerline length
- total planform area proxy
- total wetted surface area proxy
- average/min/max boundary width
- minimum centerline gap
- average hydraulic diameter
- relative parallel resistance proxy
- branch length coefficient of variation
- branch resistance-proxy coefficient of variation
- maximum curvature proxy
- middle-symmetry mean/max error
- unpaired symmetry branch count

Per-branch metrics include:

- branch ID and branch type
- source/target circle IDs
- centerline length
- top and bottom boundary lengths
- average/min/max boundary width
- planform area proxy
- cross-sectional area
- wetted perimeter
- hydraulic diameter
- wetted surface area proxy
- resistance proxy
- entry/exit tangent angle
- max/mean curvature proxy

## Resistance proxy

The resistance proxy uses:

```text
centerline_length / hydraulic_diameter^4
```

This is only a relative screening value. It intentionally omits viscosity, constants, entrance losses, two-phase effects, surface roughness, manifold coupling, and actual pressure boundary conditions.

## Symmetry handling

The recovered tributary construction is centered around a middle symmetry axis at:

```text
y = 0
```

The metrics module mirrors branches about that line and reports mean/max mirror error. This helps catch indexing mistakes where the upper and lower halves no longer match.
